** Proposed patch order

1. Cache and reuse fragment ranges in `agent-shell-ui-update-fragment`.
2. Make header/mode-line updates dirty-flag driven.
3. Skip markdown overlays when region content is unchanged.
4. Batch multiple fragment updates caused by one notification.
5. Add benchmarks/profiles for the same scenario to prove wins.

** Dialectical rule of thumb

1. If a measure only changes presentation, classify it as mitigation.
2. If a measure removes duplicated search/update work, classify it as a fix.
3. If a measure reduces both, do it first.
